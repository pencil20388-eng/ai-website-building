# SEO 优化

做官网 SEO,最关键的是先分清楚**哪些是前端页面能做的、哪些必须走后台配置、哪些要技术介入**。搞错了会白做——比如在 HTML 里写死了 SEO 信息,结果实际生效的是后台配置,前端那份只是占位。

## 一、先搞清楚 SEO 配置在哪里生效

很多正式网站的 SEO 信息(TDK:Title、Description、Keywords)是在 **admin 后台统一配置**的,前端页面里的那份只是占位、不实际生效。

所以做之前必须先确认:

- **这个网站的 SEO 在哪配?** 是后台 admin 配置,还是直接写在前端页面里?
- **如果是后台配置**:前端原型里的 title/description 等只是占位,不用纠结,真正生效的内容要在后台填,这部分通常需要技术或有后台权限的人操作。
- **如果是前端写死**:那才需要在 HTML 里认真写 SEO 信息。

不确认这一点就开始优化,很可能改了半天前端,线上根本没生效。

## 二、哪些「我们(非技术)能做」

这些是内容层面的,运营/市场自己就能把控,不需要技术介入:

1. **标题文案的关键词布局**。大标题(H1)和各级二级标题(H2)里,均匀、自然地提到:产品名、品牌词、核心场景词。不要堆砌,要读起来通顺。
2. **正文文案的可读性**。让 AI 优化文案时,核心要求是:**0 基础用户也能看懂,少用专业名词,在专业的基础上说大白话**。SEO 不是写给搜索引擎看的黑话,是写给真实用户看的——用户看得懂、愿意读,停留时间长,本身就有利于排名。
3. **H 标签层级的合理性**。一个页面只能有一个 H1(通常是主标题),其余标题按 H2、H3 往下分。层级跟着页面的视觉结构走,大部分情况下很自然就能判断——大板块标题用 H2,板块内的小标题用 H3。
4. **内容的信息结构**。哪些内容放前面、哪些卖点优先展示,这是内容策略,运营最懂。

## 三、哪些「需要技术介入」

这些偏技术,通常要开发或有后台权限的人配合,运营自己做不了或做了不生效:

1. **后台 TDK 配置**。如果 SEO 走后台,Title/Description/Keywords 的实际填写要在 admin 里做。
2. **图片 alt 文本**。每张图的 alt 描述(给搜索引擎和无障碍读屏用),前端能加,但如果图片是后台动态渲染的,要在后台配。
3. **语义化标签**。用对 HTML 标签(比如导航用 nav、主要内容用 main 等),这是开发写代码时的事,运营提需求即可。
4. **结构化数据**(Schema.org)。比如评分、产品信息的结构化标记,需要开发实现。
5. **技术性 SEO**:页面加载速度、移动端友好、URL 结构、sitemap、robots.txt 等,全部是技术侧的事。

## 四、给 AI 优化文案的指令模板

确认好 SEO 在哪生效后,让 AI 优化文案可以这样说:

```
帮我优化这个页面的标题和正文文案,要求:
1. 大标题和二级标题里自然提到:[产品名]、[品牌词]、[核心场景词]
2. 不要堆砌关键词,读起来要通顺
3. 文案让 0 基础用户也能看懂,少用专业名词,专业的基础上说大白话
4. 一个页面只保留一个 H1,其余用 H2/H3 按结构往下分
```

## 五、长期维护视角

从长期维护角度,运营和技术要分工清楚:

- **运营负责**:内容、关键词布局、文案可读性、信息结构——这些会经常调整,运营自己能改最高效。
- **技术负责**:后台 SEO 配置机制、语义化标签、结构化数据、技术性 SEO——这些建好后不常变,但需要开发一次性做对。

建议在项目早期就和技术对齐:**SEO 哪些走后台、后台怎么配、运营能自助改哪些、哪些要提需求**。这个边界划清楚,后期维护才不会每次都卡在「我改了怎么没生效」。
