# 避坑清单（开始改动前必读）

以下是真实项目中踩过的坑，每个坑后面是必须遵守的红线规则。

## 坑 1：文件丢失（最严重）

**发生了什么**：AI 在重做一个模块时，一次编辑操作把网页文件改坏了，大量内容丢失，当时没有任何备份，花了一小时从对话日志里逐条恢复。

**红线规则**：
- 每次改动前，先保存当前版本（git commit，或复制一份文件备份）。
- 一个功能 = 一次保存，不要积攒多个改动一起存。
- 这条没有例外，再小的改动也先存。

## 坑 2：在错误的地方改文件

**发生了什么**：用多个 AI/多台机器协作时，有的 AI 在自己本地的旧副本上改，改完上传反而覆盖了最新版本，反复发生。

**红线规则**：
- 始终确认「现在改的这个文件，是不是大家都在用的那一份」。
- 多方协作时，约定唯一的「正式文件」位置，所有 agent 都改那一份。
- 改之前先确认位置，不要假设会自动同步。

## 坑 3：图片压缩/换格式破坏透明背景

**发生了什么**：压缩一张带透明背景的 PNG 图时，把它转成了不支持透明的格式，结果透明区域全变成黑色，页面上一片黑。

**红线规则**：
- 带透明背景的图片（PNG）压缩或转换前，先确认它的格式和透明通道。
- 绝不把带透明背景的图转成不支持透明的格式（如 JPG）。
- 处理完图片必须在浏览器看一眼，确认没有变黑或缺失。

## 坑 4：过度设计

**发生了什么**：一个 CTA 区块，AI 一路加戏——加宇宙感背景、加渐变、加浮动光球、加旋转圆环——做了十几个版本，越加越乱，决策者一眼就觉得不专业。最后用最简洁的纯色渐变 + 文字 + 按钮、零动画解决，反而最好。

**红线规则**：
- 功能性模块优先简洁，简洁 > 花哨。
- 加任何动效/装饰前问一句「这真的需要吗」。
- 先参考真实竞品怎么做，不要凭空创造视觉。

## 坑 5：反复来回改

**发生了什么**：一个模块从左右分栏 → 对比卡片 → 浏览器窗口 → 对称布局 → 极简方案，改了 20 多版，因为每次都是盲改、没先确认方向。

**红线规则**：
- 重大改动前先问「你想要 A 还是 B」，确认方向再动手。
- 一次最多给 2 个方案对比，不要一次甩 5 个。
- 大改时直接给参考图片或模板，让 AI 照着做，比文字描述准得多。
- 方向定了才执行，不要自作主张推翻重来。

## 坑 6：误删了不可恢复的数据

**发生了什么**：排查问题时删除了一些数据，不可恢复，造成损失。

**红线规则**：
- 任何删除/覆盖/重置操作，先说清楚后果，得到同意再做。
- 不确定的时候就问，不要先斩后奏。
- 删除 = 不可恢复，当成红线对待。

## 坑 7：改了看不到效果（缓存问题）

**发生了什么**：改了代码、文件也对，但页面上还是旧的，以为没改成功，反复折腾。

**红线规则**：
- 看不到新效果时，先硬刷新：Windows 用 Ctrl+Shift+R，Mac 用 Cmd+Shift+R。
- 或者用浏览器无痕模式打开，绕过缓存。
- 确认是缓存问题后再判断「是不是真没改成功」，不要盲目重改。

## 坑 8：只顾桌面端，忘了手机端

**发生了什么**：一个模块在电脑上调得很好看，上线后用手机一看，布局错位、字太大、间距乱。因为全程只在电脑上看效果，最后才发现手机端没适配。

**红线规则**：
- 每个模块在电脑端调好后，立刻检查手机端效果。
- 移动端和桌面端同步推进，不要全做完才回头补手机端。
- 重点检查小屏上的：多列是否变单列、字号是否过大、按钮是否好点、图片是否溢出。
